# for...in Loop: Touring Object Properties

The `for...in` loop is designed for looping through the **properties** of an object. Let's explore how it works!

## The Museum Tour Guide Analogy

<KeyConcept title="Think Like a Tour Guide" icon="map">
Imagine you're a **tour guide in a museum**:

- You walk through each **room** (property) in the museum (object)
- You announce the **name of each room** (property key)
- You don't automatically describe what's inside (but you can look!)

The `for...in` loop gives you the **keys** (room names), not the values (room contents).
</KeyConcept>

<ProgressCheckpoint section="intro" xpReward={5} />

## Basic Syntax

```javascript
for (const key in object) {
  // code to run for each property
}
```

### Simple Example

```javascript
const person = {
  name: 'Alice',
  age: 25,
  city: 'New York'
};

for (const key in person) {
  console.log(key);
}
// Output: "name", "age", "city"
```

> **Notice:** We get the property *names* (keys), not the values!

## Getting Both Keys and Values

To get the value, use the key to access it:

```javascript
const car = {
  brand: 'Toyota',
  model: 'Camry',
  year: 2022
};

for (const key in car) {
  console.log(key + ': ' + car[key]);
}
// Output:
// brand: Toyota
// model: Camry
// year: 2022
```

## Keys vs Values - Understanding the Difference

| What You Get | How to Access |
|:-------------|:--------------|
| Property **key** (name) | `key` |
| Property **value** | `object[key]` |
| Both together | `key + ': ' + object[key]` |

## Important Warning: Don't Use for...in with Arrays!

While `for...in` works with arrays, it's **not recommended**:

```javascript
const colors = ['red', 'green', 'blue'];

// ❌ Not recommended - gives you indices as strings
for (const index in colors) {
  console.log(index, typeof index);
}
// Output: "0" string, "1" string, "2" string

// ✅ Better - use for...of for arrays
for (const color of colors) {
  console.log(color);
}
// Output: "red", "green", "blue"
```

<InfoBox type="warning" title="for...in vs for...of">
- **for...in** → For **objects** (gives property names)
- **for...of** → For **arrays** and iterables (gives values)

Using the wrong one is a common beginner mistake!
</InfoBox>

## Quick Reference

```javascript
const obj = { a: 1, b: 2, c: 3 };

// Get keys only
for (const key in obj) {
  console.log(key);        // "a", "b", "c"
}

// Get values only
for (const key in obj) {
  console.log(obj[key]);   // 1, 2, 3
}

// Get both
for (const key in obj) {
  console.log(`${key} = ${obj[key]}`);  // "a = 1", "b = 2", "c = 3"
}
```

> **Memory Tip:** "**in**" = "what's **in** the object" = property names!
